# LogHunter v1.8.0.1

Date: 2026-04-29

## Included in this release

- Parallelized ALB scans where per-line CPU cost justifies it: Top 50 by AVG duration, Top 50 IPs overall.
- Single-pass ALB line tokenizer for IP Summary and Status Mismatch scans (removes 3-4x repeated tokenization).
- Reverted WAF Blocked Summary and per-minute WAF chart to sequential after benchmarking.
- Lower-allocation aggregation for WAF Blocked Summary (nested IP -> URI dictionary).
- Connection-per-request, read-only SQLite handling in the ALB and IIS IP Summary viewers.
- Web layer: asset cache + ETag/304, parallel accept loop, JSON `WriteIndented` removed.

## What gets faster, what stays the same

Benchmarked on a 15GB ALB sample, single-volume cold read.

| Workflow | Result vs main 1.8.0.0 |
| --- | --- |
| Top 50 by AVG duration | ~5s faster |
| Top 50 IPs overall | ~1s faster |
| WAF Blocked Summary | unchanged (parallel revert) |
| WAF blocks per minute chart | unchanged (parallel revert) |
| IP Summary | unchanged (I/O-bound) |
| Status Mismatch and other token-heavy scans | unchanged on cold I/O |

The viewer and web layer changes are responsiveness wins (concurrent tabs, faster repeat page loads, instant stop) rather than stopwatch wins.

## Sizing rule learned

Parallel `Parallel.ForEachAsync` over input files only helps when per-line CPU work is meaningful relative to disk read cost. For substring-only filters (WAF blocked detection), parallel readers fragment OS prefetch and lose to the original sequential streamer on a single physical volume.

## Cleanup

- Removed dead `TryGetToken` helpers in `AlbIpSummaryScanner` and `AlbStatusMismatchScanner` orphaned by the single-pass tokenizer rewrite. Other copies (`AlbScanner`, `AlbLogParser`) remain in use and were left intact.
