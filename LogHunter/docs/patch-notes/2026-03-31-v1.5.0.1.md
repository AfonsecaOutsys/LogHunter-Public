# LogHunter v1.5.0.1

Date: 2026-03-31

## Included in this release

- Tightened the IIS Burst option so it favors corroborated suspicious activity over broad single-signal matches.
- Added startup update checking against `AfonsecaOutsys/LogHunter-Public`.
- Added a 6-hour snooze when the operator declines the startup update prompt.

## IIS Burst tightening

The IIS Burst option was narrowed to reduce noisy matches from ordinary clustered traffic:

- increased baseline burst thresholds
- added stronger rate and enumeration tiers
- switched burst error detection to dynamic-request 4xx behavior instead of bucket-wide 4xx totals
- added concentration/focus checks so repeated pressure on one dynamic endpoint can be surfaced more clearly
- required more corroboration between rate, enumeration, error pressure, and method mix before a bucket is promoted
- removed latency from burst severity scoring

This change is intended to improve signal quality for incident triage, even if that means some lower-confidence clustered traffic is no longer flagged.

## Startup update check

LogHunter now checks the latest public release on startup and compares it with the running version.

- update source: `AfonsecaOutsys/LogHunter-Public`
- failures stay silent so offline startup is not blocked
- the result is cached locally
- declining the prompt snoozes it for 6 hours

This keeps the behavior lightweight and operator-friendly without introducing self-update complexity.
