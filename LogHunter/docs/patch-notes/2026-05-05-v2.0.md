# LogHunter v2.0

Date: 2026-05-05

LogHunter 2.0 promotes the local web shell to a first-class operator surface and folds in the performance foundation built across the 1.6 -> 1.9 line. The console workflow remains fully supported.

## Headline: Web shell as a first-class surface

- Card-based workflow selection across ALB, IIS, Platform, and AbuseIP modules.
- Action-first layout: workflows are reachable in two seconds, no marketing copy or roadmap labels.
- Native OS file/folder pickers wired through the host process (no browser file picker friction).
- Long-running jobs run on background managers with live progress, cancel, and result handoff.
- Asset cache + ETag/304 responses, parallel accept loop, JSON without `WriteIndented` -> faster repeat page loads and snappier multi-tab use.
- Startup mode selector with remember-my-choice (Console / Web).

## Performance foundation (folded in from 1.6 -> 1.9)

- Parallel ALB scans where per-line CPU cost justifies it (Top 50 by AVG duration, Top 50 IPs overall).
- Single-pass ALB tokenizer for IP Summary and Status Mismatch (removes 3-4x repeated tokenization).
- Lower-allocation aggregation in WAF Blocked Summary (nested IP -> URI dictionaries).
- Connection-per-request, read-only SQLite handling in ALB and IIS IP Summary viewers.
- WAF Blocked Summary and per-minute WAF chart kept on the sequential reader after benchmarking showed parallel reads regressed on cold I/O.

## Coverage carried forward

- AbuseIPDB check workflow with multi-source IP selection (ALB / IIS burst / Platform suspicious cache).
- IIS Bytes Intel: Top Bandwidth IPs and URIs (`sc-bytes`), Uploads/Payload Attempts (`cs-bytes`).
- IIS Burst Patterns and IIS Status Pivot.
- Platform authenticated activity cross-reference and suspicious request extraction (CSV/XLSX).
- ALB generic scan framework: Top 50 IPs overall, Top 50 IPs by URI, Top 50 by AVG duration, WAF blocked summary.

## Runtime / packaging hygiene

- Strips the appended source-revision SHA from `AssemblyInformationalVersion` so the runtime now reports a clean `2.0.0.0` everywhere (console banner, web shell footer, `/api/app-info`, file properties).
- Single self-contained `LogHunter.exe`, signed with the LogHunter code-signing certificate.

## Compatibility

- Existing CLI flows and console commands remain available.
- Default behavior is unchanged; `--web` opts into the web shell when the startup dialog is suppressed.
